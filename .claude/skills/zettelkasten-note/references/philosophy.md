# 제텔카스텐 철학 및 콘텐츠 가이드라인

## 제텔카스텐 원칙 상세

1. **원자성 (Atomicity)**: 하나의 노트에는 하나의 아이디어만 담는다
2. **연결성 (Connectivity)**: 다른 노트와의 연결을 항상 고려한다
3. **자기 설명성**: 노트 자체만으로 내용을 이해할 수 있어야 한다
4. **고유 식별자**: 각 노트는 고유한 ID를 가진다

## 콘텐츠 풍부화 가이드라인 (Content Enrichment Guidelines)

### 상세 수준별 작성 지침

#### 요약 모드 (Summary)
사용할 섹션:
- 한 줄 요약
- 핵심 개념 (2-3문장)
- 연결된 개념 (3개 이하)
- 참고문헌

스킵할 섹션:
- 상세 설명 하위 섹션들
- 예시와 사례
- 비유와 직관
- 개인적 생각
- 타임라인

#### 보통 모드 (Standard)
사용할 섹션:
- 한 줄 요약
- 핵심 개념 (왜 중요한가 포함)
- 상세 설명 (정의, 핵심 원리, 구성 요소)
- 예시와 사례 (1-2개)
- 장단점 분석 (간략히)
- 실용적 적용 (적용 상황만)
- 생각해볼 질문 (2-3개)
- 연결된 개념 (5개 이하)
- 참고문헌

#### 상세 모드 (Detailed)
모든 섹션 사용:
- 풍부한 예시 (3개 이상)
- 상세한 비교표
- 개인적 생각 전체
- 타임라인 포함
- 추가 학습 자료
- 모든 연결 유형 (상위/하위/관련/대조)

### 풍부한 콘텐츠 작성을 위한 질문 체크리스트

노트 작성 시 다음 질문들에 답하며 내용을 풍부하게:

**이해 (Understanding)**
- [ ] 이 개념을 초등학생에게 설명할 수 있는가?
- [ ] 핵심을 한 문장으로 요약할 수 있는가?
- [ ] 왜 이것이 중요한지 설명할 수 있는가?

**맥락 (Context)**
- [ ] 이 개념이 등장한 배경은?
- [ ] 이전에는 어떻게 해결했는가?
- [ ] 어떤 문제를 해결하려고 하는가?

**구체화 (Concretization)**
- [ ] 실제 예시 3개를 들 수 있는가?
- [ ] 어떤 상황에서 사용하는가?
- [ ] 사용하면 안 되는 상황은?

**비교 (Comparison)**
- [ ] 비슷한 개념과 뭐가 다른가?
- [ ] 언제 이것을 선택해야 하는가?
- [ ] 대안은 무엇인가?

**연결 (Connection)**
- [ ] 이미 알고 있는 것과 어떻게 연결되는가?
- [ ] 상위 개념은 무엇인가?
- [ ] 하위 개념은 무엇인가?

**적용 (Application)**
- [ ] 내 상황에서 어떻게 적용할 수 있는가?
- [ ] 첫 번째 단계는 무엇인가?
- [ ] 주의할 점은 무엇인가?

**비판 (Critique)**
- [ ] 한계점은 무엇인가?
- [ ] 반론이 있다면?
- [ ] 아직 답하지 못한 질문은?

### 콘텐츠 품질 체크리스트

노트 저장 전 확인:

```
□ 핵심 개념이 명확하게 서술되어 있는가?
□ 최소 1개 이상의 구체적 예시가 있는가?
□ 실용적 적용 방법이 포함되어 있는가?
□ 기존 노트와의 연결이 있는가? (wikilinks)
□ 개인적 해석이나 생각이 담겨 있는가?
□ 출처가 명확히 기재되어 있는가?
□ 후속 탐구 질문이 있는가?
□ 태그가 적절히 부여되어 있는가?
```

### 섹션별 작성 팁

#### 핵심 개념 섹션
- **DO**: 독자가 이것만 읽어도 핵심을 파악할 수 있게
- **DO**: "왜 중요한가" 반드시 포함
- **DON'T**: 세부 사항이나 예외 케이스 언급

#### 예시와 사례 섹션
- **DO**: 구체적인 상황, 적용, 결과 구조로
- **DO**: 다양한 맥락의 예시 (최소 2개)
- **DON'T**: 추상적이거나 일반적인 예시

#### 비유와 직관 섹션
- **DO**: 독자가 이미 잘 아는 것에 빗대기
- **DO**: 한계점도 함께 언급 ("이 비유의 한계는...")
- **DON'T**: 너무 복잡하거나 낯선 비유

#### 개인적 생각 섹션
- **DO**: 솔직한 의견과 의문점
- **DO**: 기존 지식/경험과의 연결
- **DON'T**: 원본 내용을 그대로 반복

#### 연결된 개념 섹션
- **DO**: 연결 이유를 구체적으로 명시
- **DO**: 다양한 연결 유형 활용 (상위/하위/관련/대조)
- **DON'T**: 연결 이유 없이 링크만 나열

### Obsidian 콜아웃 활용

```markdown
> [!abstract] 요약
> 핵심 내용 요약

> [!tip] 팁

> [!warning] 주의
> 주의해야 할 점

> [!question] 질문
> 탐구할 질문

> [!quote] 인용
> 중요한 인용문

> [!example] 예시
> 구체적인 예시

> [!note] 노트
> 부가 설명

> [!info] 정보
> 메타 정보
```

## 노트 성숙도 관리

```yaml
status: seedling | budding | evergreen
```

**seedling (씨앗)**:
- 초기 메모, 불완전한 상태
- 핵심 개념만 있음
- 예시, 연결 부족

**budding (싹)**:
- 기본 구조 완성
- 일부 섹션 채워짐
- 연결 시작

**evergreen (상록)**:
- 완성된 노트
- 풍부한 예시와 연결
- 주기적 검토 및 업데이트
