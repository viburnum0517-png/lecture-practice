# 부서별 실습 그룹 안내

이 문서는 성우하이텍 AI 마스터 과정에서 사용하는 4개 그룹의 역할과 실습 방향을 설명합니다.
각 그룹은 자신의 업무 맥락에 맞는 데이터를 사용하고,
Claude Code를 이용해 실제로 도움이 되는 결과물을 만드는 것을 목표로 합니다.

---

## A그룹 — R&D

### 인원
- 5명

### 대표 업무
- CAE 엔지니어링 지원
- 구조/응력 해석 결과 정리
- 시험 결과 비교
- 배터리 관련 기술 검토
- 실험 결과 보고서 초안 작성

### 현업 맥락
A그룹은 수치와 기술 용어가 많고,
실험 결과나 해석 결과를 빠르게 비교해 설명해야 하는 상황이 많습니다.
특히 여러 조건의 시험 결과를 요약하거나,
복잡한 데이터를 설명 가능한 문장으로 바꾸는 일이 중요합니다.

### AI 자동화 니즈
- 복잡한 기술 문서 요약
- 조건별 결과 비교표 생성
- 시험 결과 핵심 포인트 추출
- 배터리/구조해석 메모 초안 생성
- 기술 보고서 문체 정리

### 실습 데이터 경로
- `practice-data/a-rnd/`
- 예시: 배터리 테스트 결과, 응력 해석 요약, 실험 메모, 비교표

### 기대 결과물
- 시험 결과 비교 보고서
- CAE/배터리 관련 핵심 이슈 요약 문서
- 기술 검토 메모 초안
- 발표용 1페이지 요약

---

## B그룹 — 전략

### 인원
- 3명

### 대표 업무
- 시장 분석
- 경쟁사 동향 파악
- KPI 정리
- 사업 방향 검토
- M&A 검토용 자료 초안 정리

### 현업 맥락
B그룹은 외부 자료를 읽고 핵심만 빠르게 구조화하는 능력이 중요합니다.
시장 보고서, 기사, 경쟁사 자료, 내부 KPI 메모 등을 보고
짧고 설득력 있는 문장으로 정리하는 일이 많습니다.

### AI 자동화 니즈
- 시장 보고서 요약
- 경쟁사 비교표 생성
- KPI 변동 포인트 정리
- 경영진 보고용 문장 재작성
- M&A 검토 메모 초안 생성

### 실습 데이터 경로
- `practice-data/b-strategy/`
- 예시: 시장 동향 메모, 경쟁사 비교 자료, KPI 시트, 사업 검토 문서

### 기대 결과물
- 시장/경쟁사 비교 요약
- KPI 핵심 변화 보고서
- 전략 메모 초안
- 경영진 발표용 정리 문서

---

## C그룹 — 제조

### 인원
- 6명

### 대표 업무
- 불량 분석
- 생산 관리
- 품질 이슈 정리
- 공정 개선 아이디어 도출
- 금형/현장 이슈 기록 정리

### 현업 맥락
C그룹은 현장 데이터와 이슈 로그를 바탕으로 반복 패턴을 찾는 업무가 많습니다.
불량 유형, 발생 빈도, 시간대, 설비, 공정 조건 등 여러 요소를 함께 봐야 하며,
원인 가설과 개선 방향을 빠르게 정리하는 것이 중요합니다.

### AI 자동화 니즈
- 불량 유형 자동 분류 보조
- 생산 로그 핵심 이슈 추출
- 품질 리포트 초안 생성
- 현장 메모를 보고서 문체로 정리
- 개선안 후보 정리

### 실습 데이터 경로
- `practice-data/c-manufacturing/`
- 예시: 불량 로그, 생산 실적표, 품질 메모, 설비 이슈 기록, 공정 요약

### 기대 결과물
- 불량 원인 분석 문서
- 생산/품질 일일 리포트 초안
- 개선 제안 요약표
- 현장 발표용 핵심 정리 자료

---

## D그룹 — 경영

### 인원
- 2명 + 관련 지원 인원

### 대표 업무
- 예산 검토
- 구매 비교
- 투자 검토
- 경영 판단 자료 정리
- 의사결정용 메모 작성

### 현업 맥락
D그룹은 숫자와 판단 기준을 함께 다루는 경우가 많습니다.
예산안 비교, 공급업체 비교, 투자 우선순위 판단, 비용 검토처럼
정리와 설명이 동시에 필요한 문서를 자주 만듭니다.

### AI 자동화 니즈
- 예산 비교표 요약
- 구매안 장단점 정리
- 투자 검토 메모 초안 생성
- 의사결정 포인트 정리
- 경영 보고용 문장 정돈

### 실습 데이터 경로
- `practice-data/d-management/`
- 예시: 예산안, 구매 비교표, 투자 검토 메모, 의사결정 초안

### 기대 결과물
- 예산 비교 보고서
- 구매 검토 메모
- 투자 판단 초안
- 경영진 공유용 요약 문서

---

## 그룹 공통 원칙

모든 그룹은 아래 흐름을 기본으로 실습합니다.

1. 자신의 데이터 폴더를 확인한다
2. Claude에게 부서 맥락을 설명한다
3. 원하는 결과 형식을 명확히 말한다
4. 초안을 받은 뒤 수정 지시를 한다
5. 결과물을 파일로 저장한다
6. GitHub에 올려 다시 사용할 수 있게 만든다

---

## 그룹별 발표 시 공통 질문

실습 종료 후 각 그룹은 아래 질문에 답할 수 있어야 합니다.

- 우리 부서에서 AI가 가장 빨리 도움 되는 업무는 무엇인가?
- 어떤 데이터를 넣었는가?
- 어떤 질문을 했는가?
- 결과물은 어떤 형태였는가?
- 실제 업무에 바로 적용하려면 무엇이 더 필요한가?

이 문서는 그룹별 실습의 출발점입니다.
각 그룹은 여기 적힌 방향을 바탕으로 자신들의 실제 업무 언어로 질문을 구체화하면 됩니다.
