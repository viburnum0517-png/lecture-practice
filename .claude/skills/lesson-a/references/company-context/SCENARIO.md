# 성우하이텍 AI 전환 미션

## 미션 개요

성우하이텍은 제조업 기반의 중견기업입니다.
이번 실습의 목표는 각 부서의 실무자가 AI 도구를 실제 업무에 도입하고,
짧은 시간 안에 의미 있는 결과물을 직접 만들어보는 것입니다.

이 과정은 단순한 도구 체험이 아닙니다.
“우리 부서 업무에 AI를 붙이면 어디서 바로 효과가 나는가?”를 찾고,
그 결과를 문서와 저장소 형태로 남기는 실전형 전환 미션입니다.

---

## 참여 인원

- 총 16명
- 4개 부서 그룹 참여
- 각 그룹은 서로 다른 업무 과제를 가지고 실습 진행

구성은 다음과 같습니다.

- A그룹: R&D
- B그룹: 전략
- C그룹: 제조
- D그룹: 경영

---

## MW4 목표

MW4에서는 AI 도구를 쓰기 위한 기본 환경을 만듭니다.

핵심 내용:

- Claude Code 설치 또는 실행 환경 이해
- GitHub 저장소 clone 및 열기
- 기본 질문 방식 익히기
- 부서별 데이터로 작은 분석 실습 시작
- 결과물을 파일로 저장하는 흐름 익히기

즉, MW4의 성공은 “설치가 끝남”이 아니라
“내 업무 파일을 가지고 Claude에게 질문을 시작할 수 있음”입니다.

---

## MW5 목표

MW5에서는 단순 사용을 넘어 반복 가능한 업무 방식으로 확장합니다.

핵심 내용:

- CLAUDE.md를 통한 프로젝트 컨텍스트 이해
- 부서별 작업 규칙 만들기
- 결과물을 정리하고 재사용하는 방식 익히기
- 팀 발표를 위한 구조화 연습
- 최종 결과를 GitHub에 남기기
- 필요 시 간단한 배포 경험까지 연결

---

## 회사 차원의 전환 의미

이번 실습은 개인 생산성 향상만을 위한 것이 아닙니다.
성우하이텍 내부에서 AI를 “한 번 써본 도구”가 아니라
“업무 흐름 안에 정착시키는 방식”을 배우는 첫 단계입니다.

각 부서는 자신의 현실 업무에 맞는 질문을 만들고,
결과를 검토하고,
저장하고,
공유하는 방식까지 경험해야 합니다.

---

## 성공 기준

이번 미션의 최소 성공 기준은 아래와 같습니다.

1. 각 부서별 AI 분석 결과물 1개 이상 생성
2. 결과물을 Markdown 또는 정리 문서 형태로 저장
3. GitHub에 결과물을 올려 팀이 다시 볼 수 있게 만들기

권장 성공 기준은 아래와 같습니다.

- 실무형 질문 템플릿 1개 확보
- 부서별 반복 가능한 분석 흐름 1개 확보
- 발표 가능한 수준의 요약 문서 1개 확보

---

## 실습이 끝났을 때 기대하는 변화

- 수강생이 Claude Code를 두려워하지 않는다
- 업무 데이터를 보고 어떤 질문을 해야 할지 감이 생긴다
- 결과를 그냥 읽는 것이 아니라 수정 지시를 할 수 있다
- 좋은 결과를 저장하고 GitHub에 올리는 습관이 생긴다
- 각 부서가 “AI를 어디에 붙이면 효과가 나는지” 첫 사례를 확보한다

이 미션의 핵심은 완벽한 자동화가 아닙니다.
작지만 실제로 쓸 수 있는 첫 결과물을 만드는 것입니다.
