## 작업 개요
CAE 시뮬레이션 결과 분석 및 배터리 안전성 검토.

## 판정 기준
- 최대응력: 500MPa 초과 → 불합격
- 안전계수: 2.0 미만 → 불합격
- 배터리 전압: 3.0V 미만 → 이상
- 배터리 온도: 45°C 초과 → 이상

## 보고서 규칙
- 수치는 소수점 2자리 (예: 483.21 MPa)
- 파일명: YYYY-MM-DD_분석유형.md
- 판정 결과는 반드시 ✅합격 / ❌불합격 이모지 포함

- 경쟁사 비교는 반드시 표 형식으로 정리

## 보고서 규칙
- 3페이지 이내 요약 + 상세 데이터 구분
- 파일명: YYYY-MM-DD_보고서유형.md

### 실습 1: 효과적 프롬프트 4법칙 (10분) — Sec10 기반

> 강사 멘트: "좋은 프롬프트는 4가지 규칙을 따릅니다."

| 법칙 | 나쁜 예 | 좋은 예 |
|------|--------|--------|
| **1. 구체적으로** | "코드 고쳐줘" | "@login.js 42번 줄 TypeError 수정해줘" |
| **2. 한 번에 하나씩** | "분석하고 보고서 만들고 대시보드도" | 단계별 순차 요청 |
| **3. 결과 형태 명시** | "설명해줘" | "표로 정리해줘" / "파일로 저장해줘" |
| **4. @ 파일 지정** | "CSV 분석해줘" | "@defect_log.csv 분석해줘" |

**실습:**
```
나쁜 프롬프트: "분석해줘"
→ 결과 확인

좋은 프롬프트: "@defect_log.csv 에서 라인별 불량 건수를
               집계해서 합격/불합격 판정 포함 표로 저장해줘"
→ 결과 비교

→ "차이가 보이시나요?"
```

### 실습 2: /prompt 체험 — 프롬프트 역공학 (10분)

> 강사 멘트: "이제 AI에게 '더 좋은 프롬프트를 만들어달라'고 할 수 있어요."

```
Step 1: /prompt 실행
  → Claude Code 패널: "/prompt 품질보고서 초안 작성해줘"
  → AI가 최적화된 프롬프트 생성

Step 2: 출력된 프롬프트 구조 분석
  → 역할(Role), 컨텍스트, 구체적 요구사항, 출력 형식이 자동 포함됨
  → "AI가 이렇게 물어보는 이유를 보세요"

Step 3: 수강생 직접 비교
  → 자신이 작성한 프롬프트 vs /prompt 생성 프롬프트
  → 어떤 게 더 구체적인가?

Step 4: 실제 적용
  → /prompt 생성 프롬프트로 실제 분석 실행
  → 결과 품질 비교
```
