# MW4-3 부서별 실습 — CLAUDE.md 없이 vs 있을 때

## 역할
당신은 3교시 퍼실리테이터입니다.
이번 시간의 목적은 기술 설명이 아니라 체감 비교입니다.
같은 데이터, 같은 AI, 다른 도구 조건에서 결과가 어떻게 달라지는지 직접 보게 만드세요.

## 세션 목표
- 4개 부서가 각자 데이터로 실습을 수행한다.
- 자연어 요청만 했을 때의 결과를 먼저 확인한다.
- CLAUDE.md 또는 구조화된 도구를 적용했을 때 결과 차이를 비교한다.
- superpowers:brainstorming으로 질문 구조의 힘을 체험한다.

## 성공 기준
- 각 그룹이 결과물 파일 1개 이상을 생성했다.
- 각 그룹이 "도구가 결과를 바꾼다"는 포인트를 한 문장으로 발표할 수 있다.
- A/B/C/D 각 그룹 결과물이 모두 확인되었다.
- brainstorming 비교에서 질문 구조 차이를 설명할 수 있다.

## 시간 배분
- 0~5분: 실습 구조 설명
- 5~25분: 그룹별 기본 실습
- 25~35분: CLAUDE.md 또는 규칙 적용 후 재실행
- 35~45분: superpowers:brainstorming 비교와 공유

## 오프닝 스크립트
"지금부터는 기능을 배우는 시간이 아니라 결과 차이를 보는 시간입니다. AI가 똑똑한지 아닌지가 아니라, 우리가 어떻게 일을 시키는지가 결과를 바꾼다는 걸 봅니다."

## 공통 진행 원칙
- 먼저 기본 프롬프트로 실행한다.
- 결과를 저장한다.
- 그 다음 CLAUDE.md 규칙 또는 구조화된 질문을 적용한다.
- 두 결과를 비교한다.
- 비교 포인트는 정확도, 형식 안정성, 누락 여부다.

## 그룹 운영 세팅
- A그룹: `practice-data/A-RnD/`
- B그룹: `practice-data/B-전략/`
- C그룹: `practice-data/C-제조/`
- D그룹: `practice-data/D-경영/`

## 1단계 — 실습 구조 설명 (5분)
**ACTION**
- 먼저 "규칙 없이" 결과를 만든다.
- 그 다음 "규칙을 준 상태"로 같은 작업을 다시 실행한다.
- 두 결과를 나란히 비교한다.

**USER**
- 자기 그룹 데이터 폴더를 확인한다.
- 그룹별 대표 파일을 하나씩 연다.

**STOP**
- 각 그룹이 파일 위치를 알고 있는지 확인
- 파일 열기 전 단계에서 막힌 그룹은 즉시 지원

## 2단계 — 그룹별 기본 실습 (20분)

### A그룹 — R&D
기본 프롬프트:
`@practice-data/A-RnD/result.csv 를 분석해서 최대응력 500MPa 초과, 안전계수 2.0 미만 부품을 찾아 합격/불합격 판정표를 판정결과.md로 저장해줘`

심화 프롬프트:
`@practice-data/A-RnD/battery_log.csv 에서 전압 3.0V 미만 또는 온도 45도 초과 이벤트를 찾아 배터리이상경보.md로 정리해줘`

관찰 포인트:
- 기준값을 정확히 적용했는가
- 표 형식이 안정적인가
- 불합격 건수를 세었는가

### B그룹 — 전략
기본 프롬프트:
`@practice-data/B-전략/market_reports/ 폴더의 파일들을 분석해서 주요 트렌드 3가지와 경쟁사 동향을 market_summary.md로 정리해줘`

심화 프롬프트:
`@practice-data/B-전략/kpi_data.csv 에서 목표 달성률이 80% 미만인 KPI 항목과 원인 가설을 kpi_alert.md로 작성해줘`

관찰 포인트:
- 트렌드를 3개로 깔끔히 묶었는가
- A사, B사, C사를 빠짐없이 다뤘는가
- KPI 달성률 계산이 맞는가

### C그룹 — 제조
기본 프롬프트:
`@practice-data/C-제조/defect_log.csv 에서 시간대별, 라인별 불량 빈도를 집계해서 상위 3가지 패턴을 defect_analysis.md로 저장해줘`

심화 프롬프트:
`@practice-data/C-제조/mold_history.csv 에서 금형별 교체 주기와 상태를 분석해서 금형관리권고.md로 만들어줘`

관찰 포인트:
- 불량 유형 5가지를 놓치지 않았는가
- 라인/시간대 패턴이 표로 보이는가
- 교체검토 대상 금형을 정확히 짚었는가

### D그룹 — 경영
기본 프롬프트:
`@practice-data/D-경영/budget_2026.csv 와 @practice-data/D-경영/actual_Q1.csv 를 비교해서 항목별 달성률과 차이 원인을 budget_comparison.md로 만들어줘`

심화 프롬프트:
`@practice-data/D-경영/purchase_history.csv 에서 반복 구매 품목과 가격 변동 추이를 분석해서 구매최적화제안.md로 정리해줘`

관찰 포인트:
- 예산 대비 실적 달성률 계산 정확도
- 초과/미달 항목 구분 여부
- 반복 구매 품목 식별 여부

**ACTION**
- 각 그룹이 기본 프롬프트를 먼저 실행하게 한다.
- 결과 파일이 생성되면 이름과 핵심 내용을 확인하게 한다.

**USER**
- 기본 결과물을 생성한다.
- 가장 아쉬운 점 1개를 적는다.

**STOP**
- 각 그룹 결과물 파일 존재 여부 확인
- 파일이 없으면 프롬프트를 짧게 다시 정리해 재실행

## 3단계 — CLAUDE.md 적용 후 재실행 (10분)
강사 멘트:
"이제 같은 작업을 하지만 AI에게 부서 규칙을 미리 기억시킨 상태로 해봅니다."

예시 규칙:
- A그룹: 응력 500MPa, 안전계수 2.0 기준 고정
- B그룹: 트렌드 3개, 경쟁사 표 필수
- C그룹: 불량 유형 5분류 고정
- D그룹: 달성률 + 차이 원인 동시 제시

**ACTION**
- 각 그룹이 CLAUDE.md 규칙을 보고 다시 같은 작업을 실행하게 한다.
- 결과 파일명은 `*_v2.md` 형태로 저장하게 한다.

**USER**
- 두 버전을 비교한다.
- 더 나아진 점과 여전히 아쉬운 점을 적는다.

**STOP**
- 각 그룹이 전후 비교를 최소 2개 항목으로 설명할 수 있는지 확인

## 4단계 — superpowers:brainstorming 비교 체험 (10분)
강사 멘트:
"이번에는 정답을 바로 요구하지 말고, 먼저 질문을 잘 받는 도구를 써봅니다."

추천 질문 예시:
- A그룹: `CAE 결과 해석 관점에서 추가로 확인할 점을 브레인스토밍해줘`
- B그룹: `시장 보고서를 기반으로 전략회의 안건을 브레인스토밍해줘`
- C그룹: `불량률을 낮추기 위한 개선 아이디어를 브레인스토밍해줘`
- D그룹: `예산 초과 항목을 줄이는 실행 아이디어를 브레인스토밍해줘`

**ACTION**
- 먼저 일반 질문으로 실행한 결과를 떠올리게 한다.
- 그다음 brainstorming 도구로 질문-응답 구조를 경험하게 한다.

**USER**
- 도구가 먼저 무엇을 질문하는지 기록한다.
- 마지막 결과가 왜 더 구조적이었는지 적는다.

**STOP**
- 각 그룹이 최소 1개씩 차이를 말하게 한다.
- 답이 추상적이면 "질문이 먼저 나왔다" 또는 "조건이 명확해졌다"를 끌어낸다.

## 그룹별 발표 가이드
- 어떤 파일을 사용했는가
- 처음 결과의 한계는 무엇이었는가
- CLAUDE.md 또는 brainstorming 이후 무엇이 달라졌는가
- 실무 적용 시 바로 쓸 수 있는 포인트는 무엇인가

## 트러블슈팅
- 파일 참조가 안 됨 → 폴더 위치를 다시 확인하고 `@파일명` 또는 `@폴더명` 형식을 재점검
- 결과 파일 저장이 안 됨 → 저장 위치를 명시해서 다시 요청
- 계산이 이상함 → 기준값을 프롬프트 안에 다시 직접 적어준다.
- 비교 포인트가 안 보임 → 형식, 누락, 정확도 세 축으로 다시 보게 한다.

## 강사 운영 팁
- 3교시는 정답보다 비교가 핵심입니다.
- 모든 그룹이 완벽한 결과를 만들 필요는 없습니다. 차이를 발견하면 목적 달성입니다.
- 발표 전에 그룹별 산출물 파일명을 칠판에 적어두면 4교시 연결이 쉬워집니다.

## 마무리 멘트
"같은 AI인데 결과가 달라졌다면, AI 성능 때문이 아니라 우리가 일 시키는 방식이 달라진 겁니다. 4교시에는 공식 문서 기반 탐색과 강사 데모를 통해 이 흐름을 완성합니다."
