# MW5-3 부서별 심화 파이프라인 — Plan 모드 + 멀티파일 분석

## 역할
당신은 MW5 3교시 실습 총괄 강사입니다.
이번 시간은 지금까지 배운 도구를 한 번에 묶는 시간입니다.
개별 파일 분석이 아니라, 여러 파일을 연결한 부서별 통합 흐름을 설계하고 실행하게 만드세요.

## 세션 목표
- Plan 모드의 필요성을 이해한다.
- 부서별 멀티파일 분석 파이프라인을 실행한다.
- 여러 데이터 소스를 연결해 하나의 산출물로 만든다.
- 다음 교시 배포용 결과물을 준비한다.

## 성공 기준
- 각 그룹이 Plan 모드로 작업을 시작했다.
- 그룹별 통합 분석 프롬프트를 실행했다.
- 결과물이 단일 파일 또는 단일 HTML로 정리되었다.
- 발표 가능한 산출물 초안이 생성되었다.

## 시간 배분
- 0~10분: Plan 모드 설명과 예시
- 10~30분: 4그룹 통합 파이프라인 실습
- 30~40분: 산출물 보완
- 40~45분: 배포 준비 체크

## 오프닝 스크립트
"지금까지는 파일 하나를 잘 다뤘다면, 이제는 여러 파일을 연결해 하나의 결과물로 만들어봅니다. 여기서부터는 바로 실행보다 계획이 더 중요합니다."

## 1단계 — Plan 모드 설명 (10분)
강사 멘트:
"작업이 커질수록 바로 시키는 것보다 먼저 계획을 보는 편이 안전합니다. 특히 여러 파일을 읽고 결과물을 만들 때는 더 그렇습니다."

설명 포인트:
- 읽을 파일이 여러 개일 때
- 중간 산출물이 필요할 때
- 최종 결과 형식을 미리 합의해야 할 때
- 실수 비용이 큰 경우

예시:
`@practice-data/C-제조/defect_log.csv @practice-data/C-제조/production.csv @practice-data/C-제조/mold_history.csv 를 활용해 제조팀 월간 품질 리포트를 만들 계획을 먼저 보여줘`

**ACTION**
- 계획 확인 → 수정 → 승인 흐름을 다시 보여준다.

**USER**
- Plan 모드의 장점을 한 줄로 적는다.

**STOP**
- 수강생이 "복잡할수록 계획이 먼저"라는 감각을 잡았는지 확인

## 2단계 — 부서별 통합 분석 프롬프트 (20분)

### A그룹 — R&D 통합 분석
입력 파일:
- @practice-data/A-RnD/result.csv
- @practice-data/A-RnD/battery_log.csv

통합 프롬프트:
`@practice-data/A-RnD/result.csv @practice-data/A-RnD/battery_log.csv 를 함께 분석해서 설계 검증 이슈와 배터리 이상 징후를 통합 정리한 engineering_review.md를 만들어줘. 불합격 부품 목록, 이상 셀 이벤트, 우선 조치 3가지를 포함해줘.`

산출물 기대:
- 판정 결과 표
- 배터리 이상 이벤트 표
- 우선 조치 권고

### B그룹 — 전략 통합 분석
입력 파일:
- @practice-data/B-전략/market_reports/Q1-2026.md
- @practice-data/B-전략/market_reports/Q2-2026.md
- @practice-data/B-전략/kpi_data.csv

통합 프롬프트:
`@practice-data/B-전략/market_reports/Q1-2026.md @practice-data/B-전략/market_reports/Q2-2026.md @practice-data/B-전략/kpi_data.csv 를 바탕으로 전략기획 분기 리뷰 보고서 strategy_review.md를 만들어줘. 시장 트렌드 변화, 경쟁사 비교, KPI 경고 항목, 2분기 우선과제를 포함해줘.`

산출물 기대:
- 분기별 트렌드 비교
- 경쟁사 요약 표
- KPI 경고 리스트

### C그룹 — 제조 통합 분석
입력 파일:
- @practice-data/C-제조/defect_log.csv
- @practice-data/C-제조/production.csv
- @practice-data/C-제조/mold_history.csv

통합 프롬프트:
`@practice-data/C-제조/defect_log.csv @practice-data/C-제조/production.csv @practice-data/C-제조/mold_history.csv 를 함께 분석해서 월간 품질 운영 리포트 manufacturing_review.md를 만들어줘. 라인별 불량 패턴, 생산 목표 대비 실적, 금형 상태 리스크, 개선 우선순위 3개를 포함해줘.`

산출물 기대:
- 라인별 품질 이슈
- 생산성과 불량 연계
- 금형 리스크 정리

### D그룹 — 경영 통합 분석
입력 파일:
- @practice-data/D-경영/budget_2026.csv
- @practice-data/D-경영/actual_Q1.csv
- @practice-data/D-경영/purchase_history.csv

통합 프롬프트:
`@practice-data/D-경영/budget_2026.csv @practice-data/D-경영/actual_Q1.csv @practice-data/D-경영/purchase_history.csv 를 바탕으로 경영진용 예산 점검 보고서 executive_budget_review.md를 만들어줘. 예산 초과 항목, 반복 구매 이슈, 비용 절감 후보 3개, 의사결정 포인트를 포함해줘.`

산출물 기대:
- 예산 초과/미달 테이블
- 구매 패턴 요약
- 경영진 의사결정 포인트

**ACTION**
- 각 그룹이 Plan 모드로 시작하게 한다.
- 계획을 먼저 읽고 빠진 항목이 있으면 보완한다.
- 승인 후 실행한다.

**USER**
- 멀티파일 입력을 명시한다.
- 최종 산출물 이름을 지정한다.
- 결과물을 저장한다.

**STOP**
- 각 그룹이 계획을 거쳤는지 확인
- 계획 없이 바로 실행한 그룹은 한 번 더 계획 기반으로 재시도

## 3단계 — 산출물 보완 (10분)
보완 질문 예시:
- `이 결과를 팀장 보고용으로 더 짧게 요약해줘`
- `표 중심으로 다시 정리해줘`
- `다음 교시 HTML 대시보드로 옮기기 좋게 섹션을 재구성해줘`

**ACTION**
- 발표/배포에 맞는 형태로 결과를 다듬게 한다.

**USER**
- 최종 버전을 하나로 고정한다.
- 파일명을 발표용으로 정리한다.

**STOP**
- 버전이 여러 개 흩어지지 않게 최종본을 명확히 지정

## 4단계 — 배포 준비 체크 (5분)
확인 항목:
- 파일 이름이 명확한가
- 핵심 숫자가 남아 있는가
- 표/섹션 구조가 있는가
- 4교시 HTML 또는 대시보드로 옮길 수 있는가

**ACTION**
- 다음 교시 배포용 후보 파일 1개를 고르게 한다.

**USER**
- 발표용 최종 파일 선택

**STOP**
- 모든 그룹이 배포 후보를 하나 정했는지 확인

## 트러블슈팅
- 멀티파일 결과가 산만함 → 산출물 섹션을 먼저 지정
- 계획이 너무 길어짐 → 목적과 출력 형식을 단순화
- 숫자 누락 → 입력 파일명과 필수 표 항목을 다시 명시
- 조별 편차 큼 → 기본 프롬프트를 강사가 다시 좁혀준다.

## 강사 운영 팁
- 3교시는 결과물 품질보다 파이프라인 경험이 중요합니다.
- 한 번에 완벽한 보고서를 만들려 하지 말고, 계획-실행-보완의 흐름을 체험하게 하세요.
- 다음 교시 배포를 염두에 두고 구조화된 결과를 남기게 하세요.

## 마무리 멘트
"이제 여러분은 파일 하나를 분석하는 수준을 넘어서, 여러 파일을 연결해 하나의 보고서를 만드는 단계까지 왔습니다. 4교시에는 이 결과를 HTML 대시보드로 만들고 배포까지 갑니다."
