# MW4-4 docs-guide 체험 + 강사 데모 + 발표

## 역할
당신은 4교시 마무리 강사입니다.
이 시간의 목적은 세 가지입니다.
첫째, 모르면 공식 문서를 찾는 습관을 심는다.
둘째, 강사 라이브 데모로 다음 주 기대치를 만든다.
셋째, 그룹 발표를 통해 오늘의 학습을 자기 언어로 정리하게 한다.

## 세션 목표
- docs-guide를 이용해 공식 문서 기반 답변을 받아본다.
- 강사 데모로 Plan 모드와 대시보드 생성 흐름을 보여준다.
- 각 그룹이 2~3분 발표를 수행한다.
- MW5의 방향을 이해하고 과제를 인지한다.

## 성공 기준
- docs-guide 질문을 최소 1회 실행했다.
- 강사 데모에서 계획 확인 → 승인 → 결과 생성 흐름을 봤다.
- 각 그룹이 결과물과 배운 점을 발표했다.
- 마지막에 "저장해줘", "올려줘" 루프를 다시 실행했다.

## 시간 배분
- 0~10분: docs-guide 체험
- 10~25분: 강사 라이브 데모
- 25~40분: 그룹 발표
- 40~45분: MW5 예고와 과제 안내

## 오프닝 스크립트
"오늘 마지막 시간입니다. 설치하고, 플러그인을 붙이고, 부서별 데이터도 돌려봤습니다. 이제 모르면 공식 문서를 찾는 습관과, 결과를 공유하는 습관까지 연결하겠습니다."

## 진행 원칙
- 4교시는 설명보다 정리의 시간입니다.
- 기능을 더 많이 보여주기보다, 오늘 배운 흐름을 닫아주는 것이 중요합니다.
- 발표는 잘한 점보다 '실무에 어떻게 연결할지'를 말하게 합니다.

## 1단계 — docs-guide 체험 (10분)
강사 멘트:
"새 기능이 나올 때마다 저한테 물어보실 필요 없습니다. AI가 공식 문서를 직접 읽고 알려줄 수 있습니다."

추천 질문:
- `Plan 모드가 뭐야? 공식 문서에서 찾아서 설명해줘`
- `@파일 참조가 어떻게 동작하는지 공식 문서에서 찾아줘`
- `Claude Code에서 플러그인은 어떻게 쓰는지 공식 문서 기준으로 알려줘`

**ACTION**
- 질문 1개는 강사가 시연한다.
- 나머지 1개는 수강생이 직접 입력하게 한다.

**USER**
- docs-guide 질문을 1회 이상 입력한다.
- 답변이 왜 더 신뢰된다고 느끼는지 옆 사람과 이야기한다.

**STOP**
- 공식 문서 기반 답변이라는 점을 이해했는지 확인
- 단순 검색과 공식 문서 참조의 차이를 다시 강조

## 2단계 — 강사 라이브 데모 (15분)
강사 멘트:
"이제는 분석을 넘어서 결과물을 만들어보겠습니다. C그룹 데이터를 써서 HTML 대시보드를 만드는 흐름을 보여드릴게요."

시연 프롬프트:
`@practice-data/C-제조/defect_log.csv 를 분석해서 월별 불량률 추이와 가장 많이 발생하는 불량 유형 상위 5개를 인터랙티브 그래프가 포함된 HTML 대시보드로 만들어줘`

**ACTION**
1. Plan 모드를 켠다.
2. 계획을 보여준다.
3. 수강생과 함께 계획을 읽는다.
4. 승인 후 실행한다.
5. 생성된 HTML 파일을 브라우저로 연다.

강조 포인트:
- 바로 실행하지 않고 계획을 먼저 본다.
- 무엇을 만들지, 어떤 파일을 읽을지, 어떤 결과가 나올지 사전 합의한다.
- 결과물이 브라우저에서 열리면 실무 전달물이 된다.

**USER**
- 계획서에서 이해되는 점과 낯선 점을 말한다.
- 대시보드 화면에서 가장 유용한 요소를 하나 고른다.

**STOP**
- 수강생이 Plan 모드의 역할을 설명할 수 있는지 확인
- 대시보드 결과물이 단순 텍스트 요약과 다르다는 점을 짚는다.

## 3단계 — 그룹 발표 (15분)
발표 형식:
- 2~3분 발표
- 화면 공유 가능하면 결과 파일을 직접 보여줌
- 발표 항목은 고정 템플릿으로 통일

발표 템플릿:
1. 어떤 데이터로 무엇을 분석했는가
2. 가장 인상적인 결과는 무엇인가
3. 처음 결과와 나중 결과의 차이는 무엇인가
4. 현업에 바로 써볼 수 있는 장면은 무엇인가

**ACTION**
- A, B, C, D 순서로 발표를 진행한다.
- 시간 초과 시 질문을 줄이고 다음 조로 넘긴다.

**USER**
- 대표 1명이 발표
- 나머지 인원은 보완 코멘트 1개씩 준비

**STOP**
- 네 그룹 발표가 모두 끝났는지 확인
- 발표가 추상적이면 파일명, 숫자, 규칙 같은 구체 항목으로 끌어내기

## 4단계 — MW5 예고 (5분)
강사 멘트 예시:
"오늘은 Claude Code를 처음 연결하고 써봤다면, 다음 주는 '내 방식으로 길들이는' 시간입니다."

MW5 예고 포인트:
- CLAUDE.md 심화: Always / Ask / Never 구조
- 같은 모델인데 결과가 달라지는 이유: 프롬프트 vs 규칙 vs Plan 모드
- 실무 결과물을 인터넷 링크로 공유하는 배포 경험

과제 안내:
- 본인 업무 파일 1개를 Claude Code로 분석해보기
- 결과 파일을 저장하고 GitHub에 올려보기
- 다음 주 발표용으로 캡처 1장 준비

**ACTION**
- 과제를 짧고 명확하게 말한다.
- 복잡한 과제보다 재현 가능한 과제를 제시한다.

**USER**
- 자기 업무 파일 후보를 하나 정한다.
- 다음 주 발표에 가져올 예시를 메모한다.

**STOP**
- 모두가 과제를 이해했는지 확인
- 업무 파일 후보가 없는 사람은 오늘 실습 파일 재활용을 허용

## 5단계 — 저장과 올리기 루프 재확인 (마지막 2분)
강사 멘트:
"오늘 가장 중요한 습관은 두 문장입니다. 저장해줘, 올려줘. 이 두 문장이 여러분의 AI 협업 루틴을 만듭니다."

**ACTION**
- `오늘 MW4에서 한 모든 작업 저장해줘`
- `올려줘`
- GitHub에서 최종 반영 확인

**USER**
- 마지막 commit
- 마지막 push
- 브라우저 확인

**STOP**
- 저장과 올리기까지 확인되면 수업 종료

## 트러블슈팅
- docs-guide 답변이 일반 답변처럼 보임 → 공식 문서 근거가 보이는지 다시 확인
- Plan 모드가 낯설어 불안해함 → 실행 전 계획을 보는 안전장치라고 설명
- 발표를 어려워함 → 발표 템플릿 4문장을 그대로 읽게 허용
- push가 마지막에 실패 → 발표보다 저장을 우선, push는 개별 후속 지원

## 강사 운영 팁
- 4교시는 새로운 기능을 많이 보여주는 시간처럼 보이지만, 실제로는 자신감 회복 시간입니다.
- 발표 수준보다 발표 구조를 갖추게 하는 것이 중요합니다.
- 다음 주 기대감을 만들되, 오늘 배운 것을 충분히 소화했다는 느낌으로 끝내세요.

## 마무리 멘트
"오늘은 설치, 연결, 규칙, 분석, 발표까지 한 바퀴를 돌았습니다. 다음 주에는 이걸 여러분 부서 업무 방식에 맞게 더 깊게 세팅하고, 결과물을 밖으로 배포하는 단계까지 갑니다."
