## MW4 — Claude Code 실습 (Practice 01-05)

### 배운 것
- **Claude Code 설치와 연결**: VSCode 안에서 AI와 직접 대화하며 작업
- **student-profile.md**: 내 이름/부서/업무를 AI에게 알려주는 파일 → 맞춤형 답변 가능
- **플러그인**: Claude Code에 기능을 추가하는 확장 도구
- **Git & GitHub**: 파일 변경 이력 관리 + 팀 공유 ("저장해줘 → 올려줘" 루프)

### 핵심 명령어
```
"이 파일 요약해줘"
"표로 정리해줘"
"저장해줘"
"올려줘"
```

### 산출물
- `student-profile.md` — 나를 소개하는 파일
- `my-installed-tools.md` — 설치한 도구 목록
- 부서별 데이터 분석 결과 파일

## MW5 — 워크플로우와 스킬 (Practice 06-09)

### 배운 것
- **CLAUDE.md**: AI에게 주는 업무 규칙서. "항상/물어봐/절대 안 돼" 3가지 구조
- **프롬프트 4법칙**: 역할 + 맥락 + 형식 + 제약 → 결과 품질이 달라진다
- **워크플로우**: 반복 가능한 작업 순서를 만들면 매주 자동화 가능
- **서브에이전트 첫 체험**: `/deep-research`, `/using-superpowers` → 뒤에서 작은 AI 여러 명이 동시 작업

### CLAUDE.md 구조
```markdown
## Always (항상)
- 한국어로 답변
- 결과는 표 형식으로 정리

## Ask (물어보기)
- 파일을 삭제할 때

## Never (절대 안 함)
- 개인정보 포함 파일 외부 전송 금지
```

### 프롬프트 4법칙

| 법칙 | 예시 |
|------|------|
| 역할 | "너는 제조팀 품질 분석가야" |
| 맥락 | "이 CSV는 지난달 불량 데이터야" |
| 형식 | "표로 정리하고, 상위 3개만" |
| 제약 | "전문 용어는 쉽게 풀어서" |

### 산출물
- `CLAUDE.md` — 부서 맞춤 AI 규칙 파일
- `my-weekly-check.md` — 나만의 반복 체크리스트 스킬

## MW6 — vault + 서브에이전트 (Practice 11-14)

### 배운 것
- **옵시디언(Obsidian)**: AI가 읽을 수 있는 서랍장. `.md` 파일로 자료를 쌓는 지식 베이스
- **vault 구조**: `NAVIGATION.md` + `INDEX.md`로 AI가 빠르게 탐색할 수 있는 구조 설계
- **knowledge-manager-lite**: AI가 자동으로 vault에 자료를 수집·정리하는 도구
- **search-lite**: vault 안에서 빠르게 검색하는 경량 도구 (서버 불필요)
- **서브에이전트**: 큰 작업을 작은 AI 여러 명에게 나눠 시키는 방식
- **토큰 감각**: 가벼운 도구 vs 무거운 도구 선택 → 토큰(비용)을 아끼는 습관

### 도구 선택 기준

| 상황 | 쓸 도구 | 속도 |
|------|---------|------|
| 빠른 검색 1줄 답 | `/search-lite` | 3-5초 |
| 자료 수집·정리 (빠르게) | `/knowledge-manager-lite 빠르게` | 2-3분 |
| 자료 수집·정리 (꼼꼼히) | `/knowledge-manager-lite 꼼꼼히` | 5분+ |
| 복잡한 분석·문서 생성 | 직접 분석 요청 | 30초-2분 |

### 산출물
- `NAVIGATION.md` — vault 탐색 안내 파일
- `mw1-5-wiki/` — 1-5주차 자동 정리 위키
- `External-Research/` — 외부 자료 수집 결과
