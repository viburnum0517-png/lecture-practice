# 설치 파일이 깔아주는 것들 — 왜 필요한가요?

> 이 문서는 `install-lecture-windows.ps1` 인스톨러가 설치하는 프로그램과 도구를
> **비개발자 수강생** 관점에서 설명합니다.

---

## 한 줄 요약

Claude Code가 여러분 PC에서 제대로 일하려면 **도우미 프로그램**이 필요합니다.
인스톨러가 이걸 한 번에 깔아줍니다.

---

## 1. 필수 프로그램 (Phase 1)

### Git — 파일 변경 기록기

| 항목 | 설명 |
|------|------|
| **비유** | 문서의 "되돌리기(Ctrl+Z)"를 무한으로 쓸 수 있는 도구 |
| **왜 필요** | "저장해줘", "올려줘" 명령이 이 프로그램을 통해 동작합니다 |
| **수업에서** | 실습 결과를 저장하고 GitHub에 올릴 때 매번 사용 |

### Python — 프로그래밍 언어 실행기

| 항목 | 설명 |
|------|------|
| **비유** | 전자제품의 전원 어댑터. 눈에 안 보이지만 없으면 작동 안 됨 |
| **왜 필요** | PDF 읽기 도구 등 일부 확장 기능이 Python으로 만들어져 있습니다 |
| **수업에서** | 직접 쓸 일 없음. 뒤에서 자동으로 동작 |

### Node.js — JavaScript 실행기

| 항목 | 설명 |
|------|------|
| **비유** | Claude Code가 돌아가는 엔진. 자동차의 엔진과 같음 |
| **왜 필요** | Claude Code 자체가 Node.js 위에서 실행됩니다. 없으면 Claude Code가 안 깔림 |
| **수업에서** | 직접 쓸 일 없음. Claude Code를 실행하면 자동 사용 |

### jq — 데이터 정리 도구

| 항목 | 설명 |
|------|------|
| **비유** | 엑셀의 필터 기능을 터미널에서 쓸 수 있게 해주는 도구 |
| **왜 필요** | 설정 파일이나 API 응답(JSON 형식)을 읽고 처리할 때 사용 |
| **수업에서** | 직접 쓸 일 없음. 도구들이 내부적으로 사용 |

### GitHub CLI (gh) — GitHub 연결 도구

| 항목 | 설명 |
|------|------|
| **비유** | GitHub 웹사이트에 로그인하지 않고도 파일을 올리는 리모컨 |
| **왜 필요** | "올려줘" 명령으로 GitHub에 파일을 보낼 때 인증을 처리합니다 |
| **수업에서** | GitHub 로그인, 파일 올리기에 사용 |

---

## 2. Claude Code (Phase 2)

| 항목 | 설명 |
|------|------|
| **비유** | AI 비서가 여러분 PC 사무실에 출근하는 것 |
| **왜 필요** | 이 수업의 핵심 도구. 폴더 전체를 읽고, 분석하고, 결과를 만들어줍니다 |
| **수업에서** | 모든 교시에서 직접 사용. 질문하고, 분석 시키고, 저장하는 모든 작업의 중심 |

---

## 3. MCP 서버 (Phase 4)

MCP 서버는 Claude Code에 **추가 능력**을 붙여주는 플러그인입니다.
스마트폰에 앱을 깔듯, Claude Code에 기능을 추가하는 것과 같습니다.

### Playwright — 웹 브라우저 자동화

| 항목 | 설명 |
|------|------|
| **비유** | AI가 대신 웹사이트를 열어보고 내용을 가져오는 도구 |
| **왜 필요** | 웹페이지 내용을 읽거나, 스크린샷을 찍거나, 웹 기반 작업을 자동화할 때 |
| **수업에서** | 웹 데이터 수집, 결과물 미리보기 등에 활용 가능 |

### Notion — Notion 연동

| 항목 | 설명 |
|------|------|
| **비유** | Claude Code가 Notion 문서를 직접 읽고 쓸 수 있게 하는 연결선 |
| **왜 필요** | Notion에 있는 수업 자료나 회의록을 Claude Code로 바로 가져올 수 있습니다 |
| **수업에서** | 강사가 Notion으로 자료를 공유할 때 활용 |

### PDF Tools — PDF 읽기

| 항목 | 설명 |
|------|------|
| **비유** | Claude Code에게 PDF 파일을 읽는 눈을 달아주는 도구 |
| **왜 필요** | 업무 보고서, 기술 문서 등 PDF 파일을 분석할 때 필요 |
| **수업에서** | PDF 형식의 업무 자료를 분석할 때 사용 |

### NotebookLM — Google NotebookLM 연동

| 항목 | 설명 |
|------|------|
| **비유** | 여러 문서를 한꺼번에 넣고 질문할 수 있는 AI 노트북과의 연결 |
| **왜 필요** | 대량의 문서를 묶어서 질문하거나 요약할 때 활용 |
| **수업에서** | 여러 파일을 종합 분석할 때 보조 도구로 사용 가능 |

---

## 4. 설치 후 만들어지는 것들 (Phase 4 후반)

| 항목 | 위치 | 설명 |
|------|------|------|
| **Commands** | `~/.claude/commands/` | `/lesson`, `/start-mw4-1` 같은 슬래시 명령어 |
| **Skills** | `~/.claude/skills/` | 강의 진행에 필요한 레슨 스크립트와 참조 자료 |
| **Agents** | `~/.claude/agents/` | 부서별 멘토, 품질 검토 등 역할별 AI 도우미 |
| **References** | `~/.claude/reference/` | API 예시, 디자인 용어 등 참고 자료 |

## 자주 묻는 질문

### Q: 이것들을 직접 쓸 줄 알아야 하나요?
**아닙니다.** 인스톨러가 전부 자동으로 설치합니다. 여러분은 Claude Code만 사용하면 됩니다.

### Q: 설치 후 용량이 얼마나 늘어나나요?
대략 1~2GB 정도입니다. (Node.js + Python + Git + Claude Code 합산)

### Q: 나중에 지우고 싶으면?
Windows 설정 → 앱 → 각 프로그램을 개별 제거하면 됩니다.

### Q: 회사 보안 프로그램이 막으면?
강사에게 알려주세요. 대안 설치 방법(Plan B)이 준비되어 있습니다.
